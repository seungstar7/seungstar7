좋아 👍 운영체제(OS)의 **프로세스 스케줄링(scheduling)** 은 **CPU를 어떤 프로세스에 얼마나 줄지** 결정하는 핵심 기능이야.
이건 정보처리기사 실기에서 *거의 매년 1문제 이상* 나오는 중요한 단원이기도 해.

---

## 🧭 1️⃣ 스케줄링이란?

> **CPU를 어떤 프로세스에, 어떤 순서로, 얼마 동안 할당할지를 결정하는 과정**

즉, 여러 작업이 한꺼번에 실행될 때 **누가 먼저 CPU를 사용할지** 정하는 관리자야.

---

## 🧩 2️⃣ 스케줄링의 종류 (크게 3가지)

| 구분          | 시점            | 설명                           |
| ----------- | ------------- | ---------------------------- |
| **장기 스케줄링** | 작업(Job) → 메모리 | 어떤 프로그램을 실행할지 결정             |
| **중기 스케줄링** | 메모리 ↔ 보조기억장치  | 프로세스를 일시 중단(Suspend)시킬지 결정   |
| **단기 스케줄링** | CPU 할당        | 어떤 프로세스에 CPU를 줄지 결정 (시험 핵심!) |

---

## ⚙️ 3️⃣ 단기 스케줄링(=CPU 스케줄링)의 주요 알고리즘

| 알고리즘     | 약자                            | 선점 가능 여부  | 핵심 아이디어            |
| -------- | ----------------------------- | --------- | ------------------ |
| FCFS     | First Come First Serve        | ❌ 비선점     | 먼저 온 순서대로          |
| SJF      | Shortest Job First            | ❌ 비선점     | 실행시간이 짧은 순서대로      |
| SRTF     | Shortest Remaining Time First | ✅ 선점      | 남은 시간이 짧은 순서대로     |
| RR       | Round Robin                   | ✅ 선점      | 정해진 시간(타임퀀텀)씩 돌아가며 |
| Priority | -                             | ✅/❌ 모두 가능 | 우선순위가 높은 프로세스 먼저   |
| HRN      | Highest Response Ratio Next   | ❌ 비선점     | 대기시간 고려한 SJF 개선형   |

---

## 🧮 4️⃣ 계산 공식 정리

| 항목            | 공식                 | 의미              |
| ------------- | ------------------ | --------------- |
| **대기시간 (WT)** | 완료시간 - 도착시간 - 실행시간 | CPU 잡기 전 기다린 시간 |
| **반환시간 (TT)** | 완료시간 - 도착시간        | 전체 수행 시간        |
| **평균대기시간**    | 모든 프로세스 WT의 평균     | 효율성 지표          |
| **평균반환시간**    | 모든 프로세스 TT의 평균     | 전체 작업의 속도 판단용   |

## 💡 5️⃣ 각 스케줄링 예제 (쉬운 예시로!)

### 예제 데이터

| 프로세스 | 도착시간 | 실행시간 |
| ---- | ---- | ---- |
| P1   | 0    | 5    |
| P2   | 2    | 3    |
| P3   | 4    | 1    |

---

### ✅ (1) FCFS (먼저 온 순서대로)

순서: P1 → P2 → P3

| 프로세스 | 시작 | 완료 | 대기시간 | 반환시간 |
| ---- | -- | -- | ---- | ---- |
| P1   | 0  | 5  | 0    | 5    |
| P2   | 5  | 8  | 3    | 6    |
| P3   | 8  | 9  | 4    | 5    |

👉 평균 대기시간 = (0+3+4)/3 = **2.33**
👉 평균 반환시간 = (5+6+5)/3 = **5.33**

---

### ✅ (2) SJF (실행시간 짧은 순서)

도착 순서 고려해서, 짧은 것부터!

| 순서  | 실행 |   |
| --- | -- | - |
| 0~5 | P1 |   |
| 5~6 | P3 |   |
| 6~9 | P2 |   |

👉 평균 대기시간 = (0+4+1)/3 = **1.67**

---

### ✅ (3) SRTF (남은 시간 가장 짧은 것)

도중에 짧은 게 들어오면 선점됨.

| 시간대 | 실행 프로세스 |
| --- | ------- |
| 0~2 | P1      |
| 2~5 | P2      |
| 5~6 | P3      |
| 6~9 | P1      |

👉 평균 대기시간 = (4+0+1)/3 = **1.67**

---

### ✅ (4) RR (라운드 로빈, 타임퀀텀 = 2)

| 순서  | 실행       |
| --- | -------- |
| 0~2 | P1       |
| 2~4 | P2       |
| 4~6 | P3, P1 순 |
| 6~8 | P2       |
| 8~9 | P1       |

👉 평균 대기시간 = **2.33 (대략)**

---

### ✅ (5) HRN (응답률 기준)

> 응답률 = (대기시간 + 실행시간) / 실행시간

대기 오래한 프로세스에게 가산점 줌.

---

## 🧠 6️⃣ 비유로 이해하기 (유치원생 버전 🍭)

| 알고리즘         | 비유                         |
| ------------ | -------------------------- |
| **FCFS**     | 놀이기구 줄 먼저 선 순서대로 탑승        |
| **SJF**      | 시간이 적게 걸리는 아이 먼저 태워주는 놀이공원 |
| **SRTF**     | 짧은 놀이 먼저 끝낼 수 있으면 새치기 가능!  |
| **RR**       | 2분씩 놀고, 다 돌면 다시 차례 돌아옴     |
| **Priority** | VIP 손님 먼저 놀이기구 탑승          |
| **HRN**      | 오래 기다린 아이에게 우선권 줘서 공정하게 운영 |

---

## ⏱️ 7️⃣ 시간복잡도 요약

| 알고리즘     | 시간복잡도 | 비고           |
| -------- | ----- | ------------ |
| FCFS     | O(n)  | 간단한 순차 정렬    |
| SJF/SRTF | O(n²) | 실행시간 비교 및 선택 |
| RR       | O(n)  | 큐 기반 순환      |
| Priority | O(n)  | 우선순위 비교      |
| HRN      | O(n²) | 응답률 계산 필요    |

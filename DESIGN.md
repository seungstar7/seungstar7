좋아 👍 완전 기초부터 “유치원생도 이해할 수 있을 정도로” 설명해줄게.
디자인 패턴은 **“코드를 예쁘고 효율적으로 짜는 방법 공식”** 이야.
그리고 크게 3종류로 나뉘어 👇

---

## 🌱 1️⃣ 생성(Creational) 패턴

> **객체(=new로 만드는 것)를 ‘어떻게 만들까?’** 에 초점을 맞춘 패턴이야.

### 🧩 주요 패턴 5가지

| 패턴                               | 설명                                   | 쉬운 예시                                |
| -------------------------------- | ------------------------------------ | ------------------------------------ |
| **1. Singleton (싱글톤)**           | 객체를 “하나만” 만들고, 여러 곳에서 같이 써.          | 프린터는 여러 명이 써도 한 대면 충분하지!             |
| **2. Factory Method (팩토리 메서드)**  | 객체를 직접 만들지 않고 “공장(Factory)”을 통해 만들어. | 햄버거 주문하면 주방장이 알아서 만들어주는 느낌 🍔        |
| **3. Abstract Factory (추상 팩토리)** | 팩토리 여러 개를 묶어서, “제품군 전체”를 만드는 방법      | 맥도날드 vs 버거킹 세트 메뉴처럼 🍟🍔🥤           |
| **4. Builder (빌더)**              | 복잡한 객체를 “단계별로” 만들어                   | 건축가가 설계도에 따라 천천히 집을 짓는 느낌 🏠         |
| **5. Prototype (프로토타입)**         | 이미 만든 객체를 “복제”해서 새 객체를 만듦            | 문서 복사하기(Ctrl + C, Ctrl + V) 같은 개념 📄 |

## 🧱 2️⃣ 구조(Structural) 패턴

> **객체들이 서로 “어떻게 연결될까?”** 즉, 구조를 예쁘게 짜는 게 목적이야.

### 🧩 주요 패턴 7가지

| 패턴                        | 설명                         | 쉬운 예시                           |
| ------------------------- | -------------------------- | ------------------------------- |
| **1. Adapter (어댑터)**      | 서로 안 맞는 인터페이스를 연결시켜줌       | 110V 콘센트에 220V 어댑터 꽂기 🔌        |
| **2. Bridge (브리지)**       | 기능(추상)과 구현(구체)을 분리해서 따로 관리 | 리모컨 버튼(기능)과 TV 브랜드(구현) 분리 📺    |
| **3. Composite (컴포지트)**   | 전체-부분 관계를 트리 구조로 표현        | 폴더 안에 파일/폴더가 또 있는 구조 💻         |
| **4. Decorator (데코레이터)**  | 기능을 “포장하듯이” 추가             | 아이스크림에 토핑 추가 🍦🍫               |
| **5. Facade (퍼사드)**       | 복잡한 시스템을 단순한 인터페이스로 감싸줌    | 호텔 프런트 데스크가 모든 요청을 처리 🏨        |
| **6. Flyweight (플라이웨이트)** | 같은 객체를 공유해서 메모리를 절약        | “a” 글자를 수천 번 찍을 때 같은 글자 객체를 재사용 |
| **7. Proxy (프록시)**        | 대리인이 대신 접근                 | 영화관 티켓 예매 사이트가 영화관 대신 예매 관리 🎟️ |

## ⚙️ 3️⃣ 행위(Behavioral) 패턴

> **객체들 간의 “행동, 소통” 방식**을 다루는 패턴이야.

### 🧩 주요 패턴 11가지

| 패턴                                     | 설명                           | 쉬운 예시                       |
| -------------------------------------- | ---------------------------- | --------------------------- |
| **1. Chain of Responsibility (책임 연쇄)** | 여러 객체가 순서대로 요청을 처리           | 고객센터 → 매니저 → 본사 순서로 처리 📞   |
| **2. Command (커맨드)**                   | 요청을 “명령 객체”로 만들어 나중에 실행      | TV 리모컨 버튼에 “기능을 기록해둔” 느낌 📺 |
| **3. Interpreter (인터프리터)**             | 언어 문법 해석기                    | 계산기나 SQL 파서처럼 명령 해석         |
| **4. Iterator (이터레이터)**                | 컬렉션 안의 요소를 하나씩 순서대로 접근       | for-each 반복문처럼 🍀           |
| **5. Mediator (미디에이터)**                | 객체 간의 복잡한 의사소통을 “중재자”가 관리    | 단톡방 관리자처럼 💬                |
| **6. Memento (메멘토)**                   | 객체의 상태를 저장하고 복원              | 게임 세이브/로드 기능 🎮             |
| **7. Observer (옵서버)**                  | 한 객체의 변화가 여러 객체에 자동 통보       | 유튜브 구독 알림 🔔                |
| **8. State (상태)**                      | 상태에 따라 객체의 행동이 달라짐           | 문이 열림/닫힘 상태에 따라 행동 다름 🚪    |
| **9. Strategy (전략)**                   | 알고리즘을 쉽게 교체할 수 있게            | 정렬 방법을 “버블 ↔ 퀵”으로 바꾸는 느낌 🧩 |
| **10. Template Method (템플릿 메서드)**      | 큰 틀(골격)은 부모 클래스가, 세부는 자식이 결정 | 요리 레시피 골격은 같고 재료만 다름 🍳     |
| **11. Visitor (비지터)**                  | 구조를 바꾸지 않고 새로운 기능 추가         | 회사에 감사팀이 방문해서 여러 부서 점검 🏢   |

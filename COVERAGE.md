좋아 👍 완벽하게 이해할 수 있도록 **화이트박스 테스트의 커버리지 종류별 설명 + 플로우 차트 도식화 예시**로 정리해줄게.
정보처리기사 실기에서 자주 나오는 **문제 포인트 중심**으로 설명할게.

---

## 🧠 1️⃣ 문맥 정리 : 화이트박스 테스트란?

* **정의**: 프로그램의 내부 로직(제어 구조)을 보면서 테스트하는 방법
* **목적**: 코드가 논리적으로 모든 경로를 제대로 수행하는지 확인

---

## 🧩 2️⃣ 화이트박스 테스트의 커버리지 종류

| 커버리지 종류                            | 설명                                             | 예시로 이해                                     |
| :--------------------------------- | :--------------------------------------------- | :----------------------------------------- |
| **문장(Statement) 커버리지**             | 모든 **명령문이 한 번 이상 수행**되었는지 확인                   | if문이 참이든 거짓이든 **안쪽 코드 한 번씩만 실행되면 됨**       |
| **분기(Branch) 커버리지**                | 모든 **조건문의 참/거짓(분기)**이 한 번 이상 수행되었는지 확인         | if문이 **참/거짓 두 경우 다 수행**되어야 함               |
| **조건(Condition) 커버리지**             | 조건식에 포함된 **각 개별 조건이 참/거짓 모두 한 번 이상 수행**되었는지 확인 | if(A && B)일 때, A와 B 각각 참/거짓 경우를 모두 봐야 함    |
| **조건/분기(Condition/Decision) 커버리지** | 분기 + 조건 커버리지를 결합한 것                            | 분기(참/거짓)도, 조건(A,B 각각)도 모두 수행               |
| **변경 조건/결정 커버리지(MC/DC)**           | 각 조건이 **결과에 영향을 주는지** 확인                       | A가 바뀌어야 전체 결과가 바뀌는지 검증                     |
| **다중 조건(Multiple Condition)**      | 모든 **조건의 가능한 조합**을 테스트                         | A,B가 있을 때 (T,T), (T,F), (F,T), (F,F) 모두 수행 |

---

## 🔍 3️⃣ 플로우 차트 예제

다음 의사코드를 기준으로 설명할게 👇

```plaintext
if (A && B)
    X = 1;
else
    X = 2;
```

이걸 기준으로 각 커버리지를 플로우 차트로 표현하면 다음과 같아.

---

### 🧱 (1) 문장 커버리지 (Statement Coverage)

```
Start
  ↓
 [A && B ?]───True───▶ [X=1]
    │
   False
    │
    ▼
  [X=2]
    ↓
   End
```

**목표:** 모든 명령문(X=1, X=2)이 최소 한 번 실행되면 ✅ 통과

> 즉, 한 번은 참으로, 한 번은 거짓으로 실행되면 충분.

---

### 🔀 (2) 분기 커버리지 (Branch Coverage)

```
      [조건]
     /      \
  True       False
  /             \
[X=1]           [X=2]
   \             /
     \         /
        End
```

**목표:** 조건문의 **True, False 두 가지 경로** 모두 수행

> 두 번 테스트해야 함 (한 번은 참, 한 번은 거짓)

---

### ⚖️ (3) 조건 커버리지 (Condition Coverage)

조건이 2개이므로 `(A && B)` →
각각의 조건이 **T/F**가 모두 한 번 이상 되면 OK

|  A  |  B  |  결과 |
| :-: | :-: | :-: |
|  T  |  T  |  T  |
|  T  |  F  |  F  |
|  F  |  T  |  F  |
|  F  |  F  |  F  |

> 단, **모든 분기가 실행되진 않아도 됨** → 단점 존재

---

### ⚙️ (4) 조건/분기 커버리지 (Condition/Decision Coverage)

* 분기 커버리지 + 조건 커버리지 둘 다 만족해야 함
* 즉, 각 조건(A,B)이 T/F 한 번 이상 수행되고
  전체 분기(T/F)도 한 번씩 수행되어야 함

---

### 🔁 (5) 변경 조건/결정 커버리지 (MC/DC)

> 각 조건이 **결과에 실제 영향을 미치는지** 확인해야 함

예:

* (A=T,B=T) → 참
* (A=F,B=T) → 거짓
  → 이로써 **A의 변화가 결과에 영향 준다**는 것을 검증

---

### 🧩 (6) 다중 조건 커버리지 (Multiple Condition)

> 가능한 모든 조합을 테스트

|  A  |  B  |  결과 |
| :-: | :-: | :-: |
|  T  |  T  |  T  |
|  T  |  F  |  F  |
|  F  |  T  |  F  |
|  F  |  F  |  F  |

총 4가지 조합 모두 실행해야 함 ✅

---

## 💡 시험 핵심 요약

| 구분         | 테스트 횟수 | 특징                |
| :--------- | :----- | :---------------- |
| 문장 커버리지    | 최소     | 모든 문장 실행          |
| 분기 커버리지    | 중간     | 참/거짓 분기 모두 수행     |
| 조건 커버리지    | 중간     | 각 조건이 T/F 모두 수행   |
| 조건/분기 커버리지 | 많음     | 조건 + 분기 둘 다 수행    |
| MC/DC      | 많음     | 조건이 결과에 영향 주는지 확인 |
| 다중 조건      | 최대     | 모든 조합 수행          |

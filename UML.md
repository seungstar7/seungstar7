좋아👏 이제 정보처리기사 실기에서 **UML(통합 모델링 언어)** 파트를 제대로 잡아보자.
이 부분은 시험에서 거의 **매회 1~2문제**씩 출제되고,
특히 **다이어그램 종류, 관계, 기호 의미**를 묻는 문제가 자주 나와.

---

## 🧩 UML이란?

> **UML(Unified Modeling Language)**
> → 시스템을 시각적으로 표현하기 위한 **표준 모델링 언어**야.
> 쉽게 말해, “코드를 짜기 전에 그림으로 설계하는 설계도”라고 생각하면 돼.

---

## 🎨 UML이 필요한 이유

* 프로그램을 **눈으로 보면서 구조화**하기 위해
* 팀원 간 **의사소통**을 쉽게 하기 위해
* 유지보수, 분석, 설계를 **표준화**하기 위해

---

## 💡 UML의 2가지 큰 분류

| 분류                | 설명                       | 예시                     |
| :---------------- | :----------------------- | :--------------------- |
| **구조적 다이어그램(정적)** | 시스템이 “무엇으로 구성되어 있는가”를 표현 | 클래스, 객체, 컴포넌트, 배치      |
| **행위적 다이어그램(동적)** | 시스템이 “어떻게 동작하는가”를 표현     | 유스케이스, 시퀀스, 상태, 활동, 통신 |

---

## 🧱 [1] 구조적 다이어그램 종류 (정적 구조 표현)

| 다이어그램          | 설명                      | 쉽게 비유하자면               |
| :------------- | :---------------------- | :--------------------- |
| **클래스 다이어그램**  | 클래스 간의 관계를 표현           | “가족 관계도” — 누가 부모, 자식인지 |
| **객체 다이어그램**   | 실제 인스턴스 관계 표현           | “실제 등장인물들의 관계도”        |
| **컴포넌트 다이어그램** | 시스템 구성 모듈(컴포넌트) 간 관계 표현 | “기계의 부품 설계도”           |
| **배치 다이어그램**   | 물리적 배치 (서버, 클라이언트 등) 표현 | “공장 내 설비 배치도”          |

---

## ⚙️ [2] 행위적 다이어그램 종류 (동적 흐름 표현)

| 다이어그램            | 설명                  | 비유                   |
| :--------------- | :------------------ | :------------------- |
| **유스케이스 다이어그램**  | 사용자의 관점에서 시스템 기능 표현 | “식당에서 손님이 주문하는 장면”   |
| **시퀀스 다이어그램**    | 객체 간 메시지 흐름 (시간 순서) | “대화 시나리오”            |
| **커뮤니케이션 다이어그램** | 메시지 흐름 + 객체 간 연결    | “전화선 연결도”            |
| **상태 다이어그램**     | 객체의 상태 변화 표현        | “신호등의 변화 (빨강→노랑→초록)” |
| **활동 다이어그램**     | 작업(프로세스)의 흐름 표현     | “업무 순서도, 플로우차트”      |

---

## 🔗 UML 관계 (클래스 다이어그램에서 자주 출제됨)

| 관계                      | 의미                 | 기호      | 비유                     |
| :---------------------- | :----------------- | :------ | :--------------------- |
| **연관(Association)**     | 한 객체가 다른 객체를 참조    | 실선      | 친구 관계                  |
| **집합(Aggregation)**     | 포함하되 생명주기는 독립      | 빈 마름모   | “학교–학생” (학교 없어도 학생 존재) |
| **합성(Composition)**     | 포함하며 생명주기 공유       | 채워진 마름모 | “집–방” (집 없어지면 방도 없어짐)  |
| **일반화(Generalization)** | 상속 관계              | 삼각형 화살표 | “부모–자식 클래스”            |
| **의존(Dependency)**      | 한쪽이 다른 쪽 기능을 잠시 사용 | 점선 화살표  | “A가 B를 잠깐 빌려씀”         |

## 📚 시험에서 자주 나오는 포인트

| 출제 유형           | 예시 문제                                   |
| :-------------- | :-------------------------------------- |
| UML 다이어그램 종류 묻기 | “시스템의 구조를 표현하는 UML은?” → 클래스 다이어그램       |
| 관계 기호 묻기        | “채워진 마름모로 표현되는 UML 관계는?” → 합성 관계        |
| 유스케이스 구성요소      | “유스케이스 다이어그램 구성요소 3가지?” → 액터, 유스케이스, 관계 |
| 시퀀스 다이어그램 순서    | “객체 간 메시지 흐름을 시간 순서로 표현?” → 시퀀스 다이어그램   |
| 행위/구조 구분        | “다음 중 행위 다이어그램이 아닌 것은?”                 |

## 🧩 기억 암기법 (그림처럼 떠올리기)

```
[정적]
클래스(관계), 객체(예시), 컴포넌트(부품), 배치(물리위치)
→ ‘설계도’

[동적]
유스케이스(기능), 시퀀스(대화), 상태(신호등), 활동(순서도), 통신(연결)
→ ‘움직이는 그림’
```

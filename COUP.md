좋아 👍 아주 좋은 흐름이야.
**“결합도(Coupling)”와 “응집도(Cohesion)”**는
정보처리기사 **실기 필답형**에서 *매우 자주 출제되는 핵심 단골 개념*이야.

이 부분은 문장형으로 자주 나오기 때문에
✅ **이해 중심 + 암기 포인트 + 예시**로 완전히 정리해줄게.

---

## 🧩 1️⃣ 결합도(Coupling) — “모듈 간 얼마나 의존적인가?”

---

### 📖 개념

* **결합도**는 *모듈(또는 함수, 클래스) 간의 의존성 정도*를 나타내는 지표야.
* 즉, “한 모듈이 다른 모듈에 얼마나 연결되어 있나?”를 보는 거야.

➡️ **결합도가 높으면?**
→ 모듈이 서로 의존해서 수정이 어렵다 ❌

➡️ **결합도가 낮으면?**
→ 모듈이 독립적이라 유지보수가 쉽다 ✅

---

### 📊 결합도의 종류 (높음 → 낮음 순서로 외우기!)

| 결합도 종류                             | 설명                                    | 예시                       |
| ---------------------------------- | ------------------------------------- | ------------------------ |
| **내용 결합(Content)**                 | 한 모듈이 **다른 모듈 내부 코드**를 직접 참조하거나 수정    | A 모듈이 B 모듈의 지역변수를 직접 변경  |
| **공통 결합(Common)**                  | 여러 모듈이 **전역 변수(Global variable)**를 공유 | 전역변수 `g_data`를 여러 함수가 변경 |
| **외부 결합(External)**                | 모듈이 **공유된 외부 장치나 I/O, 파일 포맷** 등을 의존   | 여러 모듈이 같은 파일 포맷을 강하게 의존  |
| **제어 결합(Control)**                 | 한 모듈이 **다른 모듈의 수행 흐름(제어)을 결정**        | 인자로 `flag`를 넘겨서 분기 결정    |
| **스탬프 결합(Stamp / Data Structure)** | **필요 이상으로 많은 데이터 구조**를 전달             | 구조체 전체를 넘겨주지만 일부 필드만 사용  |
| **자료 결합(Data)**                    | 모듈 간 **필요한 데이터만 전달**                  | 매개변수로 필요한 값만 전달          |

---

✅ **암기팁:**

> 결합도는 “내공외제스자”로 외워!
> **내(내용) → 공(공통) → 외(외부) → 제(제어) → 스(스탬프) → 자(자료)**
> → 왼쪽으로 갈수록 강결합 ❌ / 오른쪽일수록 약결합 ✅

---

## 🧩 2️⃣ 응집도(Cohesion) — “모듈 안에서 얼마나 하나의 목적에 집중하는가?”

---

### 📖 개념

* **응집도**는 *하나의 모듈 내부 기능들이 얼마나 밀접하게 관련되어 있는가*를 의미해.
* 즉, “이 모듈이 얼마나 한 가지 일에 집중하는가?”

➡️ **응집도가 높으면?**
→ 모듈이 하나의 목적만 수행, 재사용성↑, 유지보수↑ ✅

➡️ **응집도가 낮으면?**
→ 여러 일을 섞어서 복잡함 ❌

---

### 📊 응집도의 종류 (낮음 → 높음 순서로 외우기!)

| 응집도 종류                      | 설명                        | 예시                              |
| --------------------------- | ------------------------- | ------------------------------- |
| **우연적 결합(Coincidental)**    | 서로 관련 없는 기능이 하나의 모듈에 포함   | `print()` 함수 안에서 입력, 계산, 출력 다 함 |
| **논리적 결합(Logical)**         | 논리적으로 비슷한 일을 묶음           | 입력/출력/파일처리 기능을 같은 모듈로           |
| **시간적 결합(Temporal)**        | 실행 시점이 같은 기능을 묶음          | 프로그램 시작 시 초기화 기능을 하나로           |
| **절차적 결합(Procedural)**      | 특정 순서로 실행되어야 하는 기능 묶음     | 순차적 처리 함수들만 묶음                  |
| **통신적 결합(Communicational)** | 같은 데이터를 사용하는 기능 묶음        | 같은 파일이나 DB 데이터를 처리하는 함수들        |
| **순차적 결합(Sequential)**      | 한 기능의 출력이 다음 기능의 입력으로 이어짐 | A의 결과가 B의 입력으로                  |
| **기능적 결합(Functional)**      | 모듈이 하나의 명확한 기능만 수행        | `calculateTax()` — 세금 계산만 수행    |

---

✅ **암기팁:**

> 응집도는 “우논시절통순기”로 외워!
> **우(우연) → 논(논리) → 시(시간) → 절(절차) → 통(통신) → 순(순차) → 기(기능)**
> → 오른쪽으로 갈수록 고응집 ✅

---

## 💡 한눈 정리

| 구분      | 좋은 방향   | 암기어                 |
| ------- | ------- | ------------------- |
| **결합도** | 낮을수록 좋다 | 내공외제스자 (왼쪽 강결합 ❌)   |
| **응집도** | 높을수록 좋다 | 우논시절통순기 (오른쪽 고응집 ✅) |

---

## 🧠 쉬운 비유로 이해하기

| 개념      | 쉬운 예시                                                                |
| ------- | -------------------------------------------------------------------- |
| **결합도** | 친구들이 서로 숙제 다 공유해서 한 명이 바꾸면 전부 오류남 ❌ → 강결합<br>각자 맡은 부분만 하는 경우 ✅ → 약결합 |
| **응집도** | 국어 선생님이 수학도 가르치는 경우 ❌ → 저응집<br>국어만 전문으로 하는 경우 ✅ → 고응집                |

---

## 💬 자주 출제되는 실기형 문제 예시

1️⃣ “결합도가 가장 낮은 형태는?” → **자료 결합**
2️⃣ “응집도가 가장 높은 형태는?” → **기능적 응집**
3️⃣ “전역 변수를 여러 모듈이 사용하는 형태의 결합은?” → **공통 결합**
4️⃣ “모듈 내에서 한 기능의 출력이 다음 기능의 입력이 되는 형태의 응집은?” → **순차적 응집**
5️⃣ “결합도는 낮을수록, 응집도는 높을수록 좋다.” → ✅ 참

## ① NAT / PAT (Network Address Translation)

**✅ 출제포인트:** 내부 사설 IP → 공인 IP로 바꾸는 장치 (공유기 역할)

### 🔹 개념

* **NAT** : 내부 IP(사설망) ↔ 외부 IP(공인망) 주소를 변환
* **PAT** (Port Address Translation, 동적 NAT) : 하나의 공인 IP에 **포트번호**로 여러 내부 IP 매핑

| 구분      | 설명                        | 예시                                 |
| ------- | ------------------------- | ---------------------------------- |
| 정적 NAT  | 내부IP ↔ 외부IP 1:1 고정        | 192.168.0.10 → 203.0.0.10          |
| 동적 NAT  | 미리 정의된 외부IP pool 중 임의 할당  |                                    |
| **PAT** | **포트번호까지 변환**하여 여러 사용자 공유 | 192.168.0.2:5000 → 203.0.0.1:30000 |

### 🔹 기출예시

> Q. 하나의 공인 IP 주소를 다수의 사설 IP에서 공용으로 사용할 수 있도록 포트 정보를 이용해 주소를 변환하는 기술은?
> → **PAT**

## ② ARP / RARP

**✅ 출제포인트:** IP와 MAC 주소의 매핑 관계

| 구분       | 풀네임                         | 기능           | 방향               |
| -------- | --------------------------- | ------------ | ---------------- |
| **ARP**  | Address Resolution Protocol | IP주소 → MAC주소 | 상위(IP) → 하위(MAC) |
| **RARP** | Reverse ARP                 | MAC주소 → IP주소 | 하위(MAC) → 상위(IP) |

### 🔹 예시

* PC가 “192.168.0.1”의 MAC주소를 모를 때:
  → “ARP Request” 브로드캐스트 → 해당 IP의 장치가 “ARP Reply”로 MAC 응답

### 🔹 기출예시

> Q. 네트워크 통신에서 IP 주소를 물리적 주소(MAC)로 변환하는 프로토콜은?
> → **ARP**

## ③ ICMP (Internet Control Message Protocol)

**✅ 출제포인트:** 네트워크 오류·제어 메시지 전달용, Ping 명령 기반

| 기능                      | 설명                  | 예시      |
| ----------------------- | ------------------- | ------- |
| Echo Request/Reply      | 연결 확인               | ping 명령 |
| Destination Unreachable | 목적지 도달 불가           |         |
| Time Exceeded           | TTL 만료 (traceroute) |         |
| Redirect                | 경로 변경 지시            |         |

### 🔹 공격유형

| 유형                         | 설명                                  |
| -------------------------- | ----------------------------------- |
| **Ping Flood**             | 다량의 Echo Request로 대상 다운             |
| **Smurf Attack**           | 브로드캐스트 주소로 Echo Request 보내 다수 응답 폭주 |
| **ICMP Redirect Spoofing** | 라우팅 테이블 조작 공격                       |

### 🔹 기출예시

> Q. 네트워크 진단 명령 ping은 어떤 프로토콜을 이용하는가?
> → **ICMP**

---

## ④ RIP (Routing Information Protocol)

**✅ 출제포인트:** 거리 벡터 기반 라우팅 프로토콜, 최대 홉 수 15

### 🔹 개념

* **라우팅 프로토콜:** 목적지까지의 최단경로를 찾아주는 알고리즘
* **RIP 특징**

    * 거리벡터 방식 (Distance Vector)
    * **최대 홉 수: 15**
    * **갱신주기: 30초**
    * 알고리즘: Bellman-Ford

### 🔹 비교

| 구분   | RIP          | OSPF     |
| ---- | ------------ | -------- |
| 방식   | 거리벡터         | 링크상태     |
| 알고리즘 | Bellman-Ford | Dijkstra |
| 최대 홉 | 15           | 제한 없음    |
| 규모   | 소규모          | 대규모      |
| 업데이트 | 주기적(30초)     | 이벤트 기반   |

### 🔹 기출예시

> Q. RIP에서 경로를 결정할 때 사용하는 척도(metric)는?
> → **Hop 수**

> Q. RIP의 최대 홉 수는?
> → **15**

## ⑤ HDLC (High-level Data Link Control)

**✅ 출제포인트:** 데이터링크 계층 프로토콜 / 프레임 구조 문제 자주 출제

### 🔹 프레임 구조

```
| Flag | Address | Control | Information | FCS | Flag |
| 01111110 | 1byte | 1byte | Nbyte | 2byte | 01111110 |
```

### 🔹 프레임 유형

| 종류          | 의미          | 특징            |
| ----------- | ----------- | ------------- |
| **I-Frame** | Information | 데이터 전송용       |
| **S-Frame** | Supervisory | 흐름제어·에러제어용    |
| **U-Frame** | Unnumbered  | 제어용 (연결/해제 등) |

### 🔹 기출예시

> Q. HDLC 프레임에서 데이터가 존재하는 프레임 유형은?
> → **I-Frame**

> Q. HDLC 프레임의 시작과 끝을 나타내는 Flag 값은?
> → **01111110**

## ⑥ ATM (Asynchronous Transfer Mode)

**✅ 출제포인트:** 고정 크기 셀(53 byte) 기반 전송, QoS 보장

| 항목   | 설명                                          |
| ---- | ------------------------------------------- |
| 전송단위 | **53 byte(= Cell)** = Header 5 + Payload 48 |
| 전송방식 | **비동기 시분할 다중화(ATDM)**                       |
| 특징   | 회선 교환 + 패킷 교환의 장점 결합, 고속통신, QoS 지원          |
| 계층   | ATM 계층, AAL 계층, 물리계층                        |

### 🔹 기출예시

> Q. ATM에서 전송단위의 크기는 몇 byte인가?
> → **53 byte**

# 1. 핵심 개념 정리 (짧고 단호하게)

* **프로토콜** = 통신 규칙.

* **포트(port)** = 한 호스트에서 여러 서비스(프로세스)를 구분하는 번호(0 ~ 65535).

    * **소켓 = IP주소 + 포트번호** → 통신의 끝점.

* **Well-known ports (0–1023)**: OS에 의해 예약된 서비스 포트(시험에 자주 출제).

* **Ephemeral ports (1024–65535)**: 클라이언트가 임시로 사용하는 포트.

* **TCP vs UDP (핵심 차이)**

    * **TCP**: 연결형, 신뢰성(재전송/순서보장), 흐름제어, 3-way handshake → 예: HTTP, FTP(control), SMTP, SSH
    * **UDP**: 비연결형, 속도중시(신뢰성 없음), 헤더작음 → 예: DNS, DHCP, SNMP, TFTP, VoIP(일부)

* **TCP 3-way handshake** (연결 수립)

    1. 클라이언트 → 서버: **SYN**
    2. 서버 → 클라이언트: **SYN+ACK**
    3. 클라이언트 → 서버: **ACK**
       → 연결 성립. (종료는 FIN/ACK 절차)

# 2. 시험에 자주 나오는 포트표 (암기 최우선)

아래는 **빈도 높은 포트**들. 왼쪽은 포트, 가운데 서비스, 오른쪽은 TCP/UDP.

|                            포트 | 서비스                   | TCP/UDP             |
| ----------------------------: | --------------------- | ------------------- |
|                            20 | FTP Data              | TCP                 |
|                            21 | FTP Control           | TCP                 |
|                            22 | SSH (Secure Shell)    | TCP                 |
|                            23 | Telnet                | TCP                 |
|                            25 | SMTP (메일 전송)          | TCP                 |
|                            53 | DNS                   | UDP (주로), TCP(영역전송) |
|                            67 | DHCP (서버)             | UDP                 |
|                            68 | DHCP (클라이언트)          | UDP                 |
|                            69 | TFTP                  | UDP                 |
|                            80 | HTTP                  | TCP                 |
|                           110 | POP3                  | TCP                 |
|                           123 | NTP                   | UDP                 |
|                           143 | IMAP                  | TCP                 |
|                           161 | SNMP                  | UDP                 |
|                           162 | SNMP Trap             | UDP                 |
|                           443 | HTTPS (HTTP over TLS) | TCP                 |
|                           514 | Syslog                | UDP (종종)            |
|                          3306 | MySQL                 | TCP                 |
| 53, 119, 993, 995 등은 상황별로 출제됨 |                       |                     |

> 시험 포인트: **’포트 숫자 → 서비스’**를 묻거나, **어떤 프로토콜이 UDP인지** 물음이 자주 출제돼.

---

# 3. 자주 헷갈리는 부분 정리 (시험용 한 줄 요약)

* **DNS**: 주로 **UDP 53**, 큰 영역전송(zone transfer)은 **TCP 53**.
* **FTP**: 제어는 **TCP 21**, 데이터 전송은 **TCP 20**(Active mode).
* **HTTP vs HTTPS**: 80 / 443 (HTTPS = HTTP over TLS)
* **DHCP**: 서버 67, 클라이언트 68 (UDP)
* **SNMP**: 관리자는 **Manager**, 장비는 **Agent**. SNMP는 **UDP 161**.
* **Telnet(23)**은 보안 취약 → 대체는 **SSH(22)**

---

# 4. 암기 팁(간단·효과적)

* 짧은 문장으로 묶어서 외워:
  “**FTP 21/20, SSH 22, Telnet 23, SMTP 25, DNS 53, HTTP 80, HTTPS 443**”
  (문장으로: **21-20 SSH23 SMTP25 DNS53 HTTP80 HTTPS443** — 반복 암송)
* UDP 대표: **DNS(53), DHCP(67/68), TFTP(69), SNMP(161)** → “**D-D-T-S**” (DNS, DHCP, TFTP, SNMP)
* TCP 3-way handshake: **SYN → SYN+ACK → ACK** (구성요소 암기)

## 🧩 핵심 이론 요약 (이해 중심)

### 1️⃣ **라우터와 게이트웨이의 차이**

| 구분 | 라우터               | 게이트웨이               |
| -- | ----------------- | ------------------- |
| 역할 | 네트워크 간 **데이터 전달** | 서로 다른 **프로토콜 변환**   |
| 예시 | LAN ↔ WAN 연결      | TCP/IP ↔ 다른 프로토콜 변환 |

> 💬 **비유:**
> 라우터는 “길 안내자”, 게이트웨이는 “언어 통역사”.

---

### 2️⃣ **라우팅 방식**

| 유형     | 설명             | 특징                    |
| ------ | -------------- | --------------------- |
| 정적 라우팅 | 수동으로 경로 지정     | 고정된 경로, 작은 네트워크에 사용   |
| 동적 라우팅 | 라우터끼리 자동 정보 교환 | RIP, OSPF, BGP 등이 대표적 |

> 💬 **비유:**
> 정적 라우팅은 “지도에 직접 펜으로 표시한 길”,
> 동적 라우팅은 “자동 내비게이션 GPS”.

---

### 3️⃣ **NAT (Network Address Translation)**

* 사설 IP ↔ 공인 IP 변환
* **유형**

    * **SNAT**: 내부 → 외부로 변환 (Source NAT)
    * **DNAT**: 외부 → 내부 서버로 변환 (Destination NAT)
    * **PAT (포트 주소 변환)**: 여러 내부 IP가 하나의 공인 IP 공유

> 💬 **비유:**
> 집의 여러 사람이(사설 IP) 하나의 문(공인 IP)으로 나가는 구조.

---

### 4️⃣ **ICMP (Internet Control Message Protocol)**

* 네트워크 오류 진단용 프로토콜
* **ping** 명령어에 사용됨 (Echo Request / Echo Reply)
* 주요 메시지: Destination Unreachable, Time Exceeded 등

---

### 5️⃣ **라우팅 프로토콜**

| 프로토콜 | 특징                 | 메트릭(기준)   |
| ---- | ------------------ | --------- |
| RIP  | 거리 벡터, 최대 15홉      | 홉 수       |
| OSPF | 링크 상태, 대규모 네트워크 적합 | 비용 (Cost) |
| BGP  | 자율 시스템 간 경로 선택     | 경로 속성     |

> 💬 **비유:**
> RIP은 “가까운 길 우선”, OSPF는 “빠른 길 우선”,
> BGP는 “국가 간 경로 협정”.
